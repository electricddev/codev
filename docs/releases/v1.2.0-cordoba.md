# Codev v1.2.0 "Cordoba"

**Release Date**: 2025-12-11

## Release Theme

Named after the Great Mosque of Córdoba (Mezquita), a masterpiece of Islamic architecture in Spain. Its forest of columns and arches represents how documentation creates structure and clarity within complex systems. This release focuses on documentation, making Codev more understandable and accessible.

## Highlights

### Codev Cheatsheet (Spec 0051)

A comprehensive quick-reference guide covering:
- **Core Philosophies**: Natural language as programming language, multi-model collaboration, structured human-agent workflows
- **Core Concepts**: Protocols (SPIDER, TICK, MAINTAIN, EXPERIMENT), roles, context hierarchy
- **Tools Reference**: Quick command reference for `codev`, `af`, and `consult`

See: [codev/resources/cheatsheet.md](https://github.com/cluesmith/codev/blob/main/codev/resources/cheatsheet.md)

### Context Hierarchy

New conceptual model explaining how information flows in Codev projects:

![Context Hierarchy](https://raw.githubusercontent.com/cluesmith/codev/main/codev/resources/context-hierarchy.png)

**Key insight**: We build from the top down (system → product → project → implementation) and propagate learnings from the bottom up through reviews.

### Agent Farm Internals Documentation (Spec 0052)

Comprehensive documentation of how `af` (agent-farm) works internally:
- Port allocation system (4200-4299 per project)
- tmux integration and session management
- State management (SQLite + JSON)
- Git worktree lifecycle
- Dashboard server architecture

See the new "Agent Farm Internals" section in [codev/resources/arch.md](https://github.com/cluesmith/codev/blob/main/codev/resources/arch.md)

### `codev import` Command (Spec 0039 TICK-005)

AI-assisted protocol import from other Codev projects:

```bash
# Import from local project
codev import /path/to/other-project

# Import from GitHub
codev import github:owner/repo

# Preview without running Claude
codev import github:owner/repo --dry-run
```

**How it works:**
1. Fetches source codev/ directory
2. Spawns interactive Claude session with source and target context
3. Claude analyzes differences and recommends imports
4. User approves/rejects each suggested change

This replaces the older `codev-updater` and `spider-protocol-updater` agents.

### Dashboard Polish (Spec 0050)

UX improvements to the Agent Farm dashboard:
- Clickable title only (not entire row) for expanding projects
- TICK badges displayed in project list view
- Polls for projectlist.md creation on new projects

## Breaking Changes

- Removed `codev-updater` agent (replaced by `codev import`)
- Removed `spider-protocol-updater` agent (replaced by `codev import`)
